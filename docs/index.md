# Catálogo Interno

Camada auditável do ISPB Participants Catalog dentro do próprio `@cafeinadesign/ispb-participants`.

Este diretório documenta o catálogo derivado mantido dentro do próprio projeto. A distribuição pública principal de consumo continua sendo o pacote npm, enquanto `current/`, `snapshots/`, `schemas/`, `sources/` e `datapackage.json` preservam a trilha de auditoria e a proveniência dos artefatos.

## Nesta documentação

- [`catalog.md`](./catalog.md): visão geral do produto e da distribuição
- [`datasets.md`](./datasets.md): semântica dos quatro datasets canônicos
- [`schemas.md`](./schemas.md): contratos JSON Schema e tolerância a mudanças upstream
- [`snapshots.md`](./snapshots.md): histórico dos snapshots promovidos
- [`provenance.md`](./provenance.md): fontes oficiais, manifesto e trilha de auditoria
- [`licenses.md`](./licenses.md): separação de licenças de código e dados derivados
- [`changelog.md`](./changelog.md): mudanças publicadas a cada snapshot
